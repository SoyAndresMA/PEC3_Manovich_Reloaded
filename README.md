## Planteamiento


En este documento vamos a explicar dos casos de hibridación. Pero antes debemos aclarar los términos principales. Partimos del concepto de multimedia como juntar varios medios en un mismo espacio. La remediación es cuando un medio nuevo representa o imita a uno antiguo, pero usando la tecnología digital.

La cultura digital actual está redefiniendo nuestra definición de los medios. Según Manovich (2013), la hibridación se produce cuando diferentes medios se mezclan dentro del software y ya no funcionan como partes separadas, sino como una sola experiencia. En este proceso no solo se mezclan contenidos, sino también técnicas, datos y formas de interacción.

Desde esta perspectiva, en este trabajo se analizan dos casos de hibridación contemporánea.
- El primer caso es Microsoft Flight Simulator: es una plataforma de simulación de vuelo que combina cartografía digital, datos en tiempo real y visualización hiperrealista para crear una experiencia virtual del planeta.
- El segundo caso es Meta Horizon Worlds: es un entorno social inmersivo que mezcla mundo virtual, avatares, comunicación social y minivideojuegos en un espacio digital VR.

Estos dos ejemplos permiten ver como el software puede crear nuevos medios híbridos en la cultura digital actual.


## Re-descubriendo la hibridación: Microsoft Flight Simulator
Microsoft Flight Simulator es un simulador, pero no funciona solo como un videojuego normal. El programa usa muchos datos reales. Mapas, imágenes desde satélite, información del clima. Todo eso entra en el sistema todo el tiempo. El software lo junta y lo convierte en un mundo en tres dimensiones.
El planeta no está creado a mano. Las ciudades aparecen porque el programa lee los datos. Las montañas, los ríos, las carreteras, todo sale de información real. Muchas cosas vienen desde la nube. El simulador depende mucho de internet para poder mostrar el mundo completo.
Cuando el usuario pilota o simplemente esta dentro del juego, no solo está mirando una imagen. Está usando la cabina, los instrumentos, el mapa o la vista exterior. El vuelo, los datos y la imagen pasan juntos. No se puede separar una cosa de la otra.

### Dónde ocurre la hibridación:
La hibridación en Microsoft Flight Simulator ocurre dentro del propio software. No se ve como partes separadas. El mapa no está por un lado y el vuelo por otro. Todo pasa junto. El simulador no muestra datos y luego imagen, los datos ya son imagen.
El mapa deja de ser un mapa clásico; no es solo una vista desde arriba lo convierte en espacio. Se puede entrar dentro o volar encima o rodearlo. La cartografía funciona como mundo tridimensional. Esto no existe en los medios tradicionales.
La simulación de vuelo tampoco está aislada. No es solo controlar con un mando un avión. Está unida al clima real, al tráfico aéreo, al relieve del terreno. El avión responde a datos no a una animación fija. El software decide todo el tiempo.
Aquí no hay una suma de medios. No es mapa más simulador más visualización aqui todo funciona como una sola cosa, por eso no es multimedia (como ya explicamos). Es hibridación, el usuario no puede separar qué es dato, qué es imagen y qué es simulación. Todo ocurre al mismo tiempo dentro del programa.

Los datos están siempre presentes. No se ven como números, se ven como mundo. El clima cambia mientras vuelas, ves nubes reales, sensación de viento, efecto de lluvia, etc. Todo afecta al avión. No es un decorado fijo, tenemos información que entra al sistema y parte de ella es actual y real.

El terreno también es dato. La altura del suelo, la forma de la tierra, los edificios o ciudades, ahora el software usa esa información y la convierte en imagen. El usuario no piensa en datos, solo vuela. Pero los datos están trabajando todo el tiempo.

### Estética híbrida
La interfaz no es solo un menú. Es la cabina, los instrumentos, las pantallas o lo propios mapas dentro del avión. Todo forma parte de la experiencia. Cambiar de vista es normal:
Cabina
Exterior
Mapa
Suelo

No hay un punto claro donde empieza una cosa y termina otra. El usuario interactúa con datos sin saberlo. Eso es parte de la hibridación. No hay una sola forma de mirar. La interfaz mezcla muchas cosas:
Simulación
Visualización
Navegación
Control

Microsoft Flight Simulator es un caso muy claro de hibridación porque no se puede entender desde un solo medio. No es solo simulación. No es solo mapa. No es solo visualización. Todo está mezclado dentro del software.
Lo interesante es que el usuario no piensa en la tecnología mientras lo usa. Solo vuela pero detrás hay datos, algoritmos y sistemas que trabajan juntos. El mundo real se convierte en algo que se puede recorrer y experimentar de otra manera. No como espectador, sino desde dentro.
Este tipo de experiencia muestra bien lo que explica Manovich. El software no solo representa la realidad. La reorganiza y cambia la forma en que entendemos el espacio y la información. El planeta deja de ser un objeto lejano y se convierte en un entorno interactivo.
Por eso, Microsoft Flight Simulator podría aparecer sin problema en una nueva versión del libro de Manovich. Es un ejemplo actual de cómo el software crea nuevos medios híbridos y nuevas formas de relación con el mundo digital y físico.
